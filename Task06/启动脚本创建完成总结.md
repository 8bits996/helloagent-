## 💡 使用技巧

### 1. 快速验证服务状态

打开浏览器访问:
- http://localhost:8000/healthz → 应返回 `{"status":"ok"}`
- http://localhost:5174 → 应显示前端界面

### 2. 查看实时日志

启动后的两个命令窗口:
- **蓝色窗口（后端）**: 显示 API 调用、LLM 响应
- **紫色窗口（前端）**: 显示页面加载、编译信息

### 3. 服务窗口管理

- 服务窗口会自动最小化到任务栏
- 不要关闭窗口，否则服务会停止
- 可以从任务栏恢复窗口查看日志

### 4. 配置文件位置

修改配置后需要重启服务:
- 后端配置: `backend/.env`
- 前端配置: `frontend/.env.local`

## ⚠️ 常见问题快速解决

### Q1: 双击脚本没反应

**原因**: Python 或 Node.js 未安装

**快速检查**:
```cmd
打开命令提示符，运行:
python --version
node --version
```

**解决方案**:
- 安装 Python 3.10+
- 安装 Node.js 16+
- 确保添加到系统 PATH

---

### Q2: 端口被占用

**现象**: 启动失败，提示端口占用

**快速解决**:
```
1. 双击 "停止所有服务.bat"
2. 等待 5 秒
3. 重新启动
```

---

### Q3: 前端无法连接后端

**快速检查**:
1. 访问 http://localhost:8000/healthz
2. 如果无法访问，后端未启动成功
3. 查看后端窗口的错误信息

**常见原因**:
- `.env` 文件配置错误
- LLM API 密钥无效
- 网络连接问题

---

### Q4: 研究请求失败

**快速排查**:
```
1. 查看后端窗口错误日志
2. 确认 LLM_API_KEY 有效
3. 测试简单主题（如 "Python"）
4. 检查网络连接
```

## 🔧 脚本功能详解

### 快速启动.bat
```
功能:
- 自动启动后端和前端服务
- 美观的彩色界面
- 服务窗口自动最小化
- 自动打开浏览器

特点:
- 无需手动操作，全自动
- 视觉效果好，易于识别
- 启动速度快
```

### 启动所有服务.bat
```
功能:
- 环境检查（Python、Node.js）
- 详细的启动日志
- 错误提示和处理
- 自动打开浏览器

特点:
- 适合首次使用
- 详细的状态反馈
- 便于问题排查
```

### 停止所有服务.bat
```
功能:
- 自动查找并终止后端进程
- 自动查找并终止前端进程
- 确认服务已停止

使用时机:
- 需要彻底停止服务
- 准备重启系统
- 服务异常需要强制停止
```

### 重启所有服务.bat
```
功能:
- 先停止现有服务
- 等待进程完全退出
- 重新启动所有服务

使用时机:
- 修改了配置文件
- 修改了代码
- 服务出现异常
```

### 创建桌面快捷方式.bat
```
功能:
- 在桌面创建快捷方式
- 指向 "快速启动.bat"
- 方便日常使用
```
