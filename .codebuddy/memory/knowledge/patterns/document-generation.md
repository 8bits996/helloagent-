# 📄 大型文档生成最佳实践

> 标签：#document #generation #writing #best-practice
> 蒸馏来源：多次大型 Markdown 文档生成经验
> 创建：2026-03-21 | 验证次数：5+

---

## 核心原则

### 原则 1: 先规划后执行

**正确流程**：
```
构思架构 → 生成目录大纲 → 用户确认 → 分段撰写 → 整合校验
```

**反模式**：直接一次性生成全部内容 → 超长文件容易截断、质量难控

---

### 原则 2: 文件大小控制

| 操作 | 文件行数 | 推荐方式 |
|------|---------|---------|
| 创建新文件 | < 500 行 | `write_to_file` 一次写入 |
| 创建新文件 | ≥ 500 行 | 分章节多文件 或 分段写入 |
| 修改现有文件 | 任意 | `replace_in_file` 精准替换 |
| 大幅重写 | > 300 行 | 先读后分段 `replace_in_file` |

---

### 原则 3: 结构化组织

**大文档必备元素**：
1. **目录（TOC）** — 带锚点链接
2. **分节标题** — 使用统一的层级编号（一、1.1、1.1.1）
3. **表格总结** — 关键对比使用表格而非长段文字
4. **ASCII 架构图** — 复杂关系使用 ASCII art 可视化
5. **代码块格式** — 示例代码/配置/JSON 使用 fenced code blocks

---

### 原则 4: 多版本管理

当需要生成多个版本/变体时：
1. **提炼共性** — 先确定各版本共享的骨架
2. **标注差异** — 使用对照表明确各版本的独特部分
3. **独立文件** — 差异大时拆分为独立文件，而非塞在一个巨型文件中

## 经验数据

| 场景 | 推荐文件数 | 推荐单文件行数 |
|------|-----------|--------------|
| 单一主题报告 | 1 | 200~400 行 |
| 多版本框架 | 1 主 + N 附 | 主 300~500，附 200~400 |
| 全景式体系文档 | 按章节拆分 | 每章 150~300 行 |
