2. **启动 CodeBuddy**，打开包含本文件夹的项目

### 方式二：作为 Prompt 指令手册使用

每个文档都包含：
- ✅ **完整的 Agent 角色定义**（可直接作为 System Prompt）
- ✅ **Prompt 模板**（PM / Orchestrator / 执行层 Agent 模板）
- ✅ **协作协议**（Agent 间消息格式）
- ✅ **质量门控检查清单**（每个 Phase 的 Gate 标准）
- ✅ **模型选型建议**（按角色层级推荐 LLM）

你可以从文档中提取 Prompt 模板，用于任何支持多轮对话的 AI 工具。

### 方式三：作为团队架构设计参考

如果你正在设计自己的 Multi-Agent 系统，本框架提供：
- 🏗️ **分层架构模式**：PM → Orchestrator → GateKeeper → Phase Agent → Cross-Phase Agent
- 🔄 **反馈循环机制**：质量不达标时的回溯路径设计
- 📊 **模型选型矩阵**：按角色/层级/成本的 LLM 选择策略
- 🎯 **场景适配方法**：从通用骨架到特定领域的映射方法论

## 📐 架构核心理念

```
┌─────────────────────────────────────────────────┐
│  7 大设计原则                                     │
├─────────────────────────────────────────────────┤
│  1. 分层管控  — PM > Orchestrator > GateKeeper   │
│  2. 阶段门控  — 每Phase结束必须通过质量检查         │
│  3. 反馈循环  — 验证失败可回溯至任意前序阶段         │
│  4. 记忆管理  — 全程维护上下文与中间成果             │
│  5. 并行执行  — 同Phase内Agent可并发工作            │
│  6. 渐进交付  — 支持Phase级中间成果输出             │
│  7. 弹性扩展  — Agent角色池按需增减                 │
└─────────────────────────────────────────────────┘
```

> **核心理念**：多智能体不是"一个AI假装成多个角色"，而是通过**专业化分工、结构化协作、阶段性门控、全程质量保障**构建真正的"研究团队"。
