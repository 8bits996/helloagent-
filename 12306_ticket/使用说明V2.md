# 12306 抢票脚本使用指南 V2 - 增强版

## 🆕 V2版本新增功能

相比V1版本，V2版本增加了以下关键特性：

### ✅ 完整的错误处理和自动恢复
- **排队自动检测**：自动识别排队状态，排队结束后自动提交
- **订单提交失败重试**：订单提交失败会自动重试，最多10次
- **页面跳转失败恢复**：票被抢完或跳转失败时自动返回抢票页面继续
- **网络错误重试**：网络抖动时自动重试，不影响整体流程
- **乘客选择重试**：乘客选择不完整时自动重试，确保4人全部选中
- **持续URL监控**：自动检测页面变化，适应不同场景

### ✅ 智能重试机制
- 查询失败：自动刷新重试
- 票被抢完：自动寻找其他车次
- 提交失败：自动重试提交
- 跳转超时：自动返回查询页面
- 弹窗干扰：自动关闭弹窗继续操作

### ✅ 详细的状态追踪
- 实时显示当前页面状态
- 显示重试次数和耗时
- 详细的错误日志和恢复建议
- 成功/失败统计

## 🔧 故障排除

### 问题1：显示"达到最大重试次数"
**原因**：长时间未找到有票车次或多次失败

**解决**：
- 执行 `GrabXY.retry()` 重新开始
- 检查网络连接
- 确认起售时间是否正确

### 问题2：长时间卡在排队页面
**原因**：12306排队系统繁忙

**解决**：
- 等待脚本自动处理
- 排队会自动检测并提交
- 不要手动关闭页面

### 问题3：显示"订单提交失败"
**V2自动处理**：脚本会自动重试最多10次

**手动解决**：
- 查看错误提示
- 如需要，手动点击提交按钮
- 执行 `GrabXY.order()` 重新提交

### 问题4：页面跳转超时
**V2自动处理**：脚本会自动返回查询页面继续抢票

**手动解决**：
- 等待脚本自动恢复
- 或执行 `GrabXY.retry()` 手动重试

### 问题5：乘客选择不完整
**V2自动处理**：脚本会自动重试选择

**手动解决**：
- 确认4位乘客都在常用联系人
- 刷新页面后执行 `GrabXY.order()`

## 📊 状态说明

执行 `GrabXY.state` 可以查看详细状态：

```javascript
{
  isRunning: true,           // 是否正在运行
  currentPage: 'query',      // 当前页面类型
  retryCount: 156,           // 当前重试次数
  orderSubmitRetryCount: 2,  // 订单提交重试次数
  queueDetected: false,      // 是否检测到排队
  queueStartTime: null,     // 排队开始时间
  lastError: null,           // 最后一次错误
  successCount: 0,           // 成功次数
  startTime: 1234567890      // 开始时间戳
}
```

页面类型：
- `unknown`：未知页面
- `query`：查询页面（刷票）
- `order`：订单页面
- `queue`：排队页面
- `success`：成功页面（支付页）
- `fail`：错误页面
