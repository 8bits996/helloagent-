以下是关于AI数据相关问题的经验分享：

一、花钱方面

整体来看，随着AI发展，优质数据需求增大，部分特定领域或高质量标注数据确实有涨价趋势。省钱办法有多种，一是做好数据预处理，通过数据清洗去除噪声、异常值和重复数据，减少无效数据存储和处理成本。二是合理进行数据分割，根据数据量和任务需求确定训练集、验证集和测试集比例，避免因过度划分导致训练数据不足或浪费过多数据用于验证测试。三是利用开源数据集，许多公开平台如Kaggle、Hugging Face等提供了大量免费数据，可满足部分需求。

二、找数据方面

数据来源途径多样。自建数据适合有特定业务场景和数据积累能力的团队，如大型企业可收集自身业务运营数据。蹲点即关注公开数据平台更新，或参与数据竞赛获取数据，能获取到一些通用或特定场景数据。第三方公司则可提供专业标注数据服务，适合对数据质量有要求且自身标注能力不足的情况。推荐的渠道包括阿里天池、和鲸社区等国内平台，以及Kaggle、UCI Machine Learning Repository等国际平台，涵盖了电商、医疗、计算机视觉等多个领域的数据。

三、数据质检方面

自动化标注和人工打分各有优势，可结合使用以提高数据质量。对于大规模、规则相对明确的数据，如简单的文本分类、图像中常见物体标注，可先用自动化标注工具快速处理，再通过人工抽检审核，纠正错误标注。对于复杂数据，如医疗影像标注、复杂语义的文本标注等，人工打分能保证准确性，但成本较高，可先由人工标注少量样本训练模型，再用模型辅助自动化标注，最后人工复核，这样既能保证质量又能提高效率。

四、趣事方面

有的团队在收集图像数据时，未注意版权问题，大量下载网络图片用于训练，结果遭遇版权纠纷，耗费了大量时间和精力处理。还有团队为获取某特定场景数据，尝试自行模拟生成，结果生成的数据与真实数据分布差异较大，导致模型在实际应用中效果不佳，后来不得不重新收集真实数据，重新训练模型，浪费了不少前期工作成果。

👍 4赞 📅 10-16 21:47

因为大模型的训练一般分3个阶段，每个阶段需要的数据格式差异挺大，分类回答下预训练阶段（同样适用于增量预训练的mid-training阶段）、SFT阶段和强化学习阶段的数据获取方法，并补充数据飞轮优化、踩坑和神操作：

一、预训练阶段

1.找矿：

1.1. 采集来源：互联网数据（公开爬取Common Crawl、Wikipedia等、开源数据集）、私域知识（企业文档、业务数据，利用企业内部已有数据，避免重复采购）

1.2. 清洗环节：1）启发式规则过滤：去除格式错误、乱码等低质量数据；2）MinHash去重可减少30-40%冗余数据，降低存储和训练成本；3）数据脱敏：对敏感信息进行处理；4）FastText过滤：过滤低质量语料

2.质检：对清洗后的数据通过大模型进行质量评分,筛选高质量数据用于训练

二、微调（SFT）阶段

1.找矿：公开数据集（HuggingFace、各类开源问答数据集）、私域知识数据对

1.3. 标注方法：1)人工标注：质量高但成本高、速度慢；2)规则-数据编程提取；3)模型生成（Self-Instruction和Evol-Instruct）

三、强化学习阶段

聚焦代码、知识、数学三大领域。根据不同算法采用不同标注策略：PPO和DPO算法需要好坏数据对；GRPO算法需要设计合理的激励函数。

四、数据飞轮自动化

通过Airflow构建端到端流程：采集 → 清洗 → 标注 → 评分 → 训练 → 评测 → 反馈优化

五、趣事与踩坑
  • MinHash阈值调优：需要多次实验
  • 代码数据验证：单元测试自动化能显著提升代码数据质量
  • 模型评分校准：需要用人工抽检来校准自动评分标准
  • 数据脱敏重要性：必须在pipeline早期完成
六、神操作
  • Self-Instruction和Evol-Instruct结合使用，成本降低的同时保证数据多样性
  • 规则+模型双重评分，平衡质量和效率
  • Airflow自动化pipeline让数据处理效率提升数倍

爬虫：可以根据业务需求，定向去找相关内容进行爬取（如youtube）。这么做灵活性高，但需要遵守目标网站的robots.txt协议。目前主流网站都部署了成熟的风控系统，爬取量大的话，需要部署一些反风控策略，比如维护动态IP池、确保请求信息的一致性与拟人化，避免被封禁、溯源。

采买：找符合资质的供应商采买数据，可以规避合规风险、快速获取数据。只需要综合数据单价、供应商提供样本的代表性，采买数据与已有数据的重合度去评估成本。由于供应商通常仅提供小部分样本试标，其余数据的真实质量存在不确定性，可能买到手，发现大多数数据都有了。

蒸馏：通过大模型蒸馏，能够按需、大规模地生成定制化数据。由于大模型本身存在"幻觉"及能力受限，合成数据的质量并不可控。因此，数据获取后必须进行严格的质量评估与清洗。

我们组在数据质量评估这一块儿做了大量的工作，并构建了一套成熟的解决方案。欢迎有相关需求的同事随时交流探讨。

在花钱处理数据方面，我有三个小建议：

  1. 先清洗现有数据，用Pandas等工具去重、删无效值，很多时候 "旧数据再利用" 比买新数据划算。
  2. 小批量标注+数据增强：比如做图像识别，先人工标注1000张，再用旋转、裁剪、加噪等工具生成1万张增强数据，成本直接降为原来的十分之一。
  3. 分阶段采购：初期用低成本的粗标数据跑通模型，再针对模型误差大的部分，买高精度标注数据，避免一上来就全量买高价数据。

分享如下🌹🌹🌹：

1️⃣【花钱】💰

数据成本确实在飙升，尤其是标注数据。我们试过直接买现成数据集，但发现通用数据泛化性差，垂直领域数据又贵得离谱（比如医疗文本标注，单条成本超5元）。后来学聪明了——先爬公开数据（学术网站、政府开放平台），再用规则清洗（正则过滤无效符号/重复内容），最后用小模型预标注（比如用GPT-3.5粗标情感倾向），人工只修正模糊样本，成本直接砍半。

2️⃣【找矿】🔍

自建+第三方组合拳最稳。自建靠用户反馈（比如产品内置"反馈改进"按钮收集真实交互数据），但量慢；第三方优先选垂直领域供应商（比如金融数据找Wind合作，电商评论对接爬虫公司），比通用数据集适配度高。悄悄说：学术会议的数据集常被低估，很多论文附带的实验数据质量超高且免费，挖一挖能省大钱。

3️⃣【质检】👀

自动化打基础+人工兜底。先用规则筛明显错误（比如图片模糊/文本乱码），再用小模型跑置信度评分（低于阈值的人工复审），最后让"老法师"（经验丰富的标注员）抽查10%做质量锚点。我们团队还搞了"标注员排行榜"，准确率高的给奖金，效率和质量一起卷。

📊 六、经验总结归类

💰

花钱省钱策略

  • 数据预处理：清洗去重，减少无效数据成本
  • 分阶段采购：先粗标后精标
  • 数据增强：小批量标注+旋转裁剪，成本降为1/10
  • 司内共享：AIdata已节省1个小目标
  • 开源数据集：Kaggle、HuggingFace、UCI
🔍

数据来源渠道

  • 公开数据集：Common Crawl、Wikipedia、ImageNet
  • 爬虫采集：遵守robots.txt，注意反风控
  • 业务自建：用户反馈、产品内置收集
  • 第三方采购：专业供应商，注意合规
  • 数据合成：大模型蒸馏、GAN生成
💯

质量控制方法

  • 标注规范先行：统一标准，避免返工
  • 人机协同：AI预标注+人工复核
  • 质量评分：大模型评分+人工抽检校准
  • 数据飞轮：采集→清洗→标注→评分→训练→评测闭环
🤗

踩坑经验

  • 标注信息不一致：医生用文件名信息标注，模型只能看图片
  • 版权问题：未注意版权大量下载，遭遇纠纷
  • 合成数据偏差：模拟数据与真实分布差异大
  • 经典数据集有错：开源数据集标注本身有问题

💡 核心经验总结

  • 数据质量比数量更重要 - 1000个高质量样本可能胜过100万个噪声数据
  • 标注规范是第一步 - 即使是专家标注，也需要严格统一的标注规范
  • 信息一致性 - 确保模型看到的信息和标注者看到的信息一致
  • 构建数据飞轮 - 让产品自身能产生并消化数据，形成闭环
  • 合规性是底线 - 在项目开始前，务必厘清数据版权和隐私问题
  • 质量从源头控制 - 质量无法靠最后一道质检关卡挽回
