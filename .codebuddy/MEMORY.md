## 二、记忆体系架构（Memory Architecture）

### 三层记忆模型

```
Layer 1: 瞬时记忆 (Ephemeral)
    ├── 存储：会话上下文（不持久化）
    ├── 生命周期：单次会话
    └── 内容：中间搜索结果、临时工具调用

Layer 2: 工作记忆 (Working)
    ├── 存储：.codebuddy/memory/projects/{slug}/
    ├── 生命周期：项目生命周期
    └── 内容：项目特定决策、配置、当前状态

Layer 3: 长期记忆 (Long-term)
    ├── 存储：MEMORY.md + memory/knowledge/
    ├── 生命周期：永久（定期蒸馏更新）
    └── 内容：用户偏好、通用模式、经验教训
```

### 四个记忆域（Memory Domains）

| 域 | 路径 | 内容 | 加载时机 |
|----|------|------|---------|
| **A. 项目域** | `memory/projects/{slug}/` | 项目特定信息 | 按需（提及项目时） |
| **B. 规则域** | `rules/` | 可执行指令/模板 | 按 alwaysApply 或触发词 |
| **C. 经验域** | `memory/knowledge/` | 跨项目通用经验 | 相关场景自动推荐 |
| **D. 偏好域** | `MEMORY.md` | 用户偏好/习惯 | 每次会话自动加载 |

### 隔离规则

```
项目域之间：完全隔离（不同目录，互不访问）
项目域 → 经验域：单向蒸馏（去项目化后提炼）
经验域 → 项目域：只读推荐（不修改项目记忆）
偏好域：全局只读（所有域可访问，仅用户可修改）
规则域：按触发条件加载（不主动加载无关规则）
```

## 四、知识复利引擎（Knowledge Compounding Engine）

### 运作机制

```
使用知识/规则/框架
      │
      ▼
[追踪] 记录到 _usage_log.md
      │
      ▼
[分析] 识别改进信号
      │
      ├─ 频率阈值（≥3次同类经验）
      ├─ 效果反馈（成功/失败/部分成功）
      └─ 用户确认（明确表示值得沉淀）
      │
      ▼
[蒸馏] 去项目化 → 提炼为通用模式
      │
      ▼
[存储] 写入 knowledge/patterns/ 或 lessons/
      │
      ▼
[记录] 变更写入 _evolution_log.md
      │
      ▼
[验证] 下次使用时检验改进效果
      │
      └─ 有效 → 保留 + 增加验证次数
      └─ 无效 → 标记 + 请求用户裁决
```

### 自动进化触发条件

| 触发器 | 条件 | 动作 |
|--------|------|------|
| **频率触发** | 同类经验 ≥ 3 次 | 提炼为通用模式 |
| **确认触发** | 用户明确确认改进 | 更新对应文件 |
| **失败触发** | 规则/框架使用失败 | 标记改进候选 |
| **矛盾触发** | 新经验与旧经验矛盾 | 请求用户裁决 |
| **过期触发** | 30天未使用 | 归档到历史 |

### 公共记忆（经验域）使用范围

**允许进入经验域 ✅：**
- 通用工作模式（如「多Agent设计应先定义门控再设计角色」）
- 工具使用经验（如「PRISMA流程适合系统性文献综述」）
- 架构决策原则（如「贯穿型Agent应 ≤ 5个」）
- 质量检查清单
- 错误教训与避坑指南

**禁止进入经验域 ❌：**
- 项目特定数据（ROI数值、客户名称、具体配置）
- 敏感信息（API密钥、账号密码、Token）
- 临时状态（调试结果、搜索缓存）
- 个人隐私（未明确允许公开的信息）
- 原始文件内容（大段复制粘贴）

### 访问控制矩阵

```
          读项目域  读规则域  读经验域  读偏好域  写自身
项目域      ✅自身    ✅按需    ✅推荐    ✅自动    ✅
规则域      ❌        ✅自身    ✅改进    ✅自动    ✅(版本化)
经验域      ❌        ❌        ✅自身    ✅自动    ✅(蒸馏后)
偏好域      ❌        ❌        ✅        ✅自身    ✅(用户触发)
```

## 五、日志维护规则

1. **日志追加**：每日工作记录追加到 `memory/YYYY-MM-DD.md`，加项目标签
2. **项目分流**：日志中的关键决策同步更新到对应项目域的 `state.md`
3. **经验蒸馏**：每周检查是否有可蒸馏到经验域的新模式
4. **30天清理**：超过30天的日志蒸馏到 MEMORY.md 或经验域后归档
5. **冲突处理**：新经验与旧经验矛盾时，追加修正条目而非覆盖
