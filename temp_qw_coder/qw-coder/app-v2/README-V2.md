# 企业微信回调服务 V2 版本

## 📝 版本说明

**V2 版本** 是在 V1 基础上的增强版本，集成了 **CodeBuddy** AI 编程助手，可以自动处理企业微信消息并返回智能回复。

### 主要特性

- ✅ 企业微信消息接收与解密
- ✅ 集成 CodeBuddy AI 处理消息
- ✅ 自动跳过权限检查 (`--dangerously-skip-permissions`)
- ✅ 支持超时控制和错误处理
- ✅ 消息长度自动截断（避免超过企微限制）
- ✅ 白名单权限控制
- ✅ 详细日志记录

## 🚀 快速开始

### 1. 配置环境变量

```bash
# 复制配置示例
cp .env.v2.example .env

# 编辑配置文件
vim .env
```

### 2. 启动服务

```bash
# 方式1：使用启动脚本（推荐）
./start-v2.sh

# 方式2：直接使用 uvicorn
python3 -m uvicorn app-v2:app --host 0.0.0.0 --port 8088
```

### 3. 查看日志

```bash
tail -f app-v2.log
```

### 4. 停止服务

```bash
./stop-v2.sh
```

## 📋 配置说明

### 环境变量

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `QYWX_TOKEN` | 企业微信回调 Token | - | ✅ |
| `QYWX_ENCODING_AES_KEY` | 企业微信加密密钥 | - | ✅ |
| `PORT` | 服务监听端口 | 8088 | ❌ |
| `ALLOWED_USERS` | 白名单用户（逗号分隔） | rtx | ❌ |
| `CODEBUDDY_PATH` | CodeBuddy 路径 | /root/.nvm/.../codebuddy | ❌ |
| `CODEBUDDY_TIMEOUT` | 执行超时时间（秒） | 300 | ❌ |
| `WORKING_DIR` | CodeBuddy 工作目录 | /root/workspace/autocoder | ❌ |

### 端口说明

- **V1 版本**: 默认端口 `8088`
- **V2 版本**: 默认端口 `8088`（避免冲突）

可以通过设置 `PORT` 环境变量来修改端口。

## 🔧 工作原理

```
用户发送企微消息
    ↓
企微服务器回调
    ↓
V2 服务接收并解密
    ↓
白名单验证
    ↓
调用 CodeBuddy 处理
    ↓
获取 AI 回复
    ↓
通过企微返回给用户
```

### 消息处理流程

1. **接收消息**: 企业微信回调接口接收加密消息
2. **解密验证**: 使用 AES 解密消息内容
3. **权限检查**: 验证用户是否在白名单中
4. **调用 CodeBuddy**:
   - 使用 `--dangerously-skip-permissions` 跳过权限检查
   - 使用 `--print` 获取非交互式输出
   - 设置超时时间避免长时间等待
5. **返回结果**: 将 CodeBuddy 的回复通过企微发送给用户

## 🔍 日志说明

日志文件位于 `app-v2.log`，包含以下信息：

- **解密消息**: 接收到的原始消息
- **解析消息**: 解析后的用户信息和内容
- **调用CodeBuddy**: CodeBuddy 的调用参数
- **CodeBuddy成功/失败**: 执行结果
- **发送消息**: 发送给用户的回复内容

日志格式：
```
[2025-12-29 00:30:15] [消息类型]
详细内容
--------------------------------------------------------------------------------
```

## ⚠️ 注意事项

1. **权限跳过**: V2 版本使用 `--dangerously-skip-permissions` 参数，请确保在受信任的环境中使用
2. **超时控制**: 默认超时 300 秒（5分钟），可通过 `CODEBUDDY_TIMEOUT` 调整
3. **消息长度**: 企微限制单条消息 2048 字符，超长消息会自动截断
4. **并发处理**: 使用 FastAPI BackgroundTasks 异步处理，避免阻塞
5. **Python 版本**: 兼容 Python 3.6+，使用 uvicorn 0.15.0

## 🐛 故障排查

### 服务无法启动

1. 检查端口是否被占用：`netstat -tuln | grep 8088`
2. 检查环境变量是否正确加载：`echo $QYWX_TOKEN`
3. 查看日志：`cat app-v2.log`

### CodeBuddy 调用失败

1. 检查 CodeBuddy 路径：`ls -la $CODEBUDDY_PATH`
2. 测试 CodeBuddy：`codebuddy --version`
3. 检查工作目录权限

### 消息未回复

1. 检查白名单配置
2. 查看日志中的错误信息
3. 测试 webhook 地址是否可访问
