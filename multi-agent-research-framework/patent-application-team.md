## 一、概述与设计理念

### 1.1 为什么需要专利申请 Multi-Agent 团队

专利申请是一项高度**跨学科、流程严谨、法规密集**的专业工作：

| 挑战维度 | 具体表现 |
|---------|---------|
| **专业复合性** | 需同时具备技术理解、法律知识、商业洞察 |
| **法规动态性** | 专利法、实施细则、审查指南持续修订（2026年1月起29处新修改生效） |
| **格式严谨性** | 权利要求书、说明书、附图均有严格格式规范 |
| **策略复杂性** | 涉及申请类型选择、同日双申、PCT、优先权等多维策略 |
| **批量规模化** | 企业级专利布局常需数十件甚至上百件并行推进 |
| **对抗博弈性** | 需模拟审查员视角预判驳回理由，提前准备答复 |

### 1.2 核心设计原则

```
┌──────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────┤
```

## 二、2024—2026 年专利法规重大变化速览

> ⚠️ **当前时间：2026 年 3 月 21 日**，以下法规均已生效实施。

### 2.1 2024 年 1 月 20 日——专利法实施细则修改（历次最大）

新增 30 条、删除 4 条，共 13 章 149 条：

| 序号 | 修改领域 | 核心内容 |
|-----|---------|---------|
| ① | 完善申请制度 | 优化流程、细化局部外观设计、放宽新颖性宽限期、完善优先权 |
| ② | 完善审查制度 | 规范申请行为、**新增延迟审查制度**、完善复审 |
| ③ | 加强行政保护 | **专利权期限补偿制度**、完善纠纷处理 |
| ④ | 加强公共服务 | **开放许可制度**、完善职务发明 |
| ⑤ | 促进转化运用 | 提升信息公共服务能力 |

### 2.2 2026 年 1 月 1 日——专利审查指南修改（局令第 84 号，29 处修改）

#### 🔴 创造性判断标准三大核心变化

| 变化 | 内容 | 对撰写的影响 |
|-----|------|------------|
| **整体评价** | 创造性判断以权利要求限定的**完整技术方案**为评估核心 | 权利要求必须体现完整技术方案 |
| **贡献入权** | 贡献性特征**必须明确写入权利要求书** | ❗仅说明书记载而未入权的不被纳入创造性评价 |
| **三步法细化** | 最接近现有技术优先关联技术问题；问题表述不含区别特征暗示；公知常识需有依据 | 技术问题表述更精准 |

#### 🔴 其他重要修改

- **一案两请（同日双申）机制调整**：进一步细化操作流程
- **发明人规定**：必须是自然人，填写身份信息并确保真实，**不得填写AI系统名称**
- **补交实验数据**：需为从原申请公开内容中可得到的
- **期限计算**：法定休假日（非仅节假日）后第一个工作日
- **权利恢复**：程序简化

### 2.3 AI 相关发明专利政策（2025 年）

- 国家知识产权局发布《人工智能相关发明专利申请指引》
- **明确 AI 参与的发明创造可以申请专利保护**
- AI 辅助发明：对实质性特点作出创造性贡献的**自然人**可署名为发明人
- AI 系统及其他非自然人**不得作为发明人**

## 四、管控层详细设计

### 4.1 PM Agent（专利总监）

```yaml
角色: 专利总监 / Patent Director
职责:
  - 确定企业专利战略方向（进攻型/防御型/平衡型）
  - 审批专利布局方案和预算
  - 评审重大专利决策（PCT、核心专利等）
  - 最终验收专利申请文件质量
  - 处理Phase间升级的决策事项
```

### 4.2 Orchestrator Agent（专利项目调度器）

```yaml
角色: 专利项目调度器
职责:
  - 将专利挖掘结果分配到各撰写Agent
  - 管理多件专利申请的并行推进
  - 协调Agent间的依赖关系
  - 处理批量申请的优先级排序
调度策略:
  - Phase 0 内按依赖链串行
  - Phase 1 内大部分可并行
  - Phase 2 内 A13→A14→A15 严格串行（权利要求优先）
  - Phase 3 内 A19~A24 全部并行
  - 多件专利间可按Phase级别并行
```

## 五、Phase 0：专利挖掘与创新捕获

> **目标**：从技术方案中系统性识别可专利的创新点，评估专利性。

### A1 InnovationCaptureAgent — 创新捕获师

```yaml
职责: 从多种来源识别可专利的创新点
输入源: 技术方案文档/产品需求/论文/代码/会议纪要
输出: 创新点清单（技术主题、技术问题、技术方案、有益效果）
关键能力: 区分「已知技术组合」与「真正创新」
```

### A2 TechDecomposeAgent — 技术分解师

```yaml
职责: 将复杂技术方案分解为多个可独立保护的技术点
分解维度: 方法层面/装置层面/系统层面/应用层面
策略: 核心技术→独立发明 | 改进→独立或合并 | 外围→围栏 | 防御→阻止规避
```

### A3 PriorArtSearchAgent — 现有技术检索师

```yaml
职责: 全面检索现有技术
检索数据库: CNIPA、USPTO、EPO/Espacenet、JPO、WIPO/PatentScope、Google Patents、学术数据库
检索策略: 关键词检索+分类号检索+引用追踪+申请人追踪+语义检索
输出: 检索报告（检索式+相关文献列表+相似度评级+技术对比表）
```

### A4 NoveltyAssessAgent — 新颖性预评估师

```yaml
职责: 基于检索结果评估新颖性（专利法第22条第2款）
方法: 单独对比原则，逐一比对技术特征，标注区别特征
输出: 新颖性评级 A(高)/B(中)/C(低)/D(丧失)
```

### A5 InventivenessAssessAgent — 创造性预评估师

```yaml
职责: 依据2026年最新「三步法」标准评估创造性
2026新规适配:
  - 第一步: 优先考虑与技术问题关联的现有技术
  - 第二步: 技术问题不含对区别特征的暗示
  - 第三步: 公知常识需有工具书依据
  - 关键: 贡献性特征是否已计划纳入权利要求
输出: 创造性评级 A+(极高)/A(高)/B(中)/C(低)/D(无)
```

### A6 PatentMiningReportAgent — 挖掘报告生成师

```yaml
职责: 汇总评估结果，生成《专利挖掘报告》
报告结构: 技术概述→创新点清单→检索摘要→专利性评级汇总→建议申请方案→布局建议→风险预警
```

### 🚧 Gate 0：挖掘质量门控

```
□ 创新点清单是否完整覆盖技术方案
□ 检索是否覆盖主要数据库（至少 CNIPA+USPTO+EPO）
□ 是否有至少1个评级为A的创新点（否则建议终止）
□ 挖掘报告是否完整
```

## 六、Phase 1：专利布局与策略规划

> **目标**：制定最优专利布局方案，保护范围最大化、成本最优化。

### A7 PortfolioStrategyAgent — 专利组合策略师

```yaml
布局模式:
  路障式: 申请必需方案，阻塞竞争者必经之路（成本低，需确认不可绕开）
  城墙式: 所有规避设计方案全部申请，不留替代空间（适用多种解决方案的技术）
  地毯式: 系统覆盖每个步骤，形成专利地雷区（成本高，适合大企业）
  围栏式: 核心+改进+应用场景，形成保护层（推荐，平衡成本与保护）
组合设计:
  核心专利（1~3件）→ 改进专利（3~8件）→ 外围专利（5~15件）→ 防御专利（2~5件）
```

### A8 FilingTypeAgent — 申请类型决策师

```yaml
类型选择矩阵:
  创造性A+/A + 长生命周期 → 发明专利
  创造性A+/A + 短生命周期 → 同日双申（发明+实用新型）
  创造性B + 任意          → 同日双申
  创造性C + 任意          → 仅实用新型
  涉及外观 + 任意         → 外观设计（含局部外观）
同日双申（2026新规适配）:
  - 同日提交发明和实用新型
  - 发明授权前选择：放弃实用新型 或 修改范围使其不同
```

### A9 GeographyAgent — 地域策略师

```yaml
路径: PCT路径（30/31个月国家阶段）| 巴黎公约（12个月优先权）| 直接申请
目标市场选择: 主要市场国+制造国+竞争对手所在地+许可目标地
```

### A10 TimingAgent — 时机策略师

```yaml
策略: 提前公开（缩短周期）| 延迟审查（最长3年）| 优先审查（6-12个月）| PPH加快
关键时限: 优先权日→12/6个月 | PCT→30/31个月 | 实审请求→3年内
```

### A11 CostBudgetAgent — 费用预算师

```yaml
费减策略: 个人减85% | 小微企业减70%
费减范围: 申请费、实审费、年费（前6年）、复审费
```

### A12 CompetitorPatentAgent — 竞争对手专利分析师

```yaml
输出: 竞品专利布局热力图 + FTO自由实施分析 + 规避设计建议 + 无效资源储备
```

### 🚧 Gate 1：策略质量门控

```
□ 布局方案覆盖核心技术各维度  □ 申请类型有充分依据
□ 同日双申符合2026新规        □ 费用在预算范围
□ FTO风险已识别               □ 批量优先级已排序
```

## 七、Phase 2：专利文件撰写

> ⚠️ 核心 Phase，直接决定授权率和保护范围。

### A13 ClaimArchitectAgent — 权利要求架构师

```yaml
设计模板:
  权利要求1（独立-方法）→ 权利要求2~4（从属）
  权利要求5（独立-装置）→ 权利要求6~7（从属）
  权利要求8（独立-系统）
  权利要求9（独立-存储介质）
  权利要求10（独立-电子设备）
2026关键: 贡献性特征必须纳入权利要求
```

### A14 ClaimWriterAgent — 权利要求撰写师

```yaml
格式:
  独立权: "一种[技术主题]，其特征在于，包括：[技术特征]"
  从属权: "根据权利要求X所述的[技术主题]，其特征在于，所述[特征]…"
禁止表述: "等等"、"例如"、"约"、"优选"、"发明人发现"
```

### A15 SpecWriterAgent — 说明书撰写师

```yaml
五段式结构:
  [技术领域] 本发明涉及...技术领域，尤其涉及一种...
  [背景技术] 现有技术描述→现有技术不足→引用文献
  [发明内容] 技术问题→技术方案（与独权一致）→有益效果（≥3个）
  [附图说明] 图1为...示意图；
  [具体实施方式] 实施例1（必须，对应独权）+实施例2（推荐）+实施例3（可选）
```

### A16 DrawingAgent — 附图设计师

```yaml
类型: 流程图/架构图/结构图/电路图/界面示意图/实验结果图
规范: 黑白线条、标记统一编号、尺寸≥4cm×4cm、引线不交叉
```

### A17 AbstractAgent — 摘要撰写师

```yaml
要求: ≤300字 | 含发明名称+方案要点+用途 | 无商业宣传 | 选摘要附图
```

### A18 RequestFormAgent — 请求书填写师

```yaml
2026新规:
  - 发明人必须填写所有身份信息并确保真实
  - 不得填写AI系统名称
  - 同日申请声明（如适用）
```

### 🚧 Gate 2：撰写质量门控

```
□ 权利要求结构完整（独权+从属，方法+装置+系统）
□ 独权保护范围合理  □ 贡献性特征已入权【2026关键】
□ 说明书五段式完整  □ 实施例充分（≥1个）
□ 附图规范且与说明书对应  □ 摘要≤300字
□ 术语全文一致  □ 发明人均为自然人【2026关键】
```

## 十四、CNIPA 标准格式模板

### 14.1 权利要求书模板

```
1. 一种[技术方案名称]的方法，其特征在于，包括以下步骤：
   步骤S1：[第一步操作描述]；
   步骤S2：基于所述[S1的结果]，[第二步操作描述]；
   步骤S3：根据所述[S2的结果]，[第三步操作描述]，得到[最终结果]。

2. 根据权利要求1所述的方法，其特征在于，所述步骤S1中，[进一步限定]。

3. 根据权利要求1所述的方法，其特征在于，所述步骤S2包括：
   步骤S21：[子步骤]；步骤S22：[子步骤]。

4. 一种[技术方案名称]的装置，其特征在于，包括：
   [模块A]，用于[功能]；[模块B]，与所述[模块A]连接，用于[功能]。

5. 一种电子设备，包括存储器和处理器，所述存储器存储有计算机程序，
   其特征在于，所述处理器执行所述计算机程序时实现权利要求1-3
   任一项所述方法的步骤。

6. 一种计算机可读存储介质，其上存储有计算机程序，其特征在于，
   所述计算机程序被处理器执行时实现权利要求1-3任一项所述方法的步骤。
```

### 14.2 说明书模板

```
[技术领域] 本发明涉及[大领域]技术领域，尤其涉及一种[具体主题]。

[背景技术] 现有技术描述→不足指出→(可选)文献引用

[发明内容]
  技术问题: 为解决上述问题，本发明提供了...
  技术方案: （与独权一致的完整描述）
  有益效果: （至少3个具体可验证的效果）

[附图说明] 图1为...示意图；图2为...示意图

[具体实施方式] 实施例1（完整，对应独权）+ 实施例2（变体）+ 实施例3（可选）
```

## 十六、批量申请调度策略

### 优先级公式

```
优先级 = 紧急度(0.4) × 价值度(0.4) × 就绪度(0.2)

紧急度: P0竞争风险/即将公开 > P1核心技术 > P2改进 > P3防御
价值度: V-S战略级 > V-A高价值 > V-B一般 > V-C低价值
就绪度: R-A完全就绪 > R-B基本就绪 > R-C部分就绪 > R-D未就绪
```

### 批量节奏

```
第1批(W1-2): 核心 3-5件（P0+P1, V-S+V-A）
第2批(W3-4): 改进 5-8件（P1+P2, V-A+V-B）
第3批(W5-8): 外围 8-15件（P2+P3, V-B+V-C）
持续:        防御专利随技术进展提交

并行度: Phase2撰写3-5件 | Phase3审核5-8件 | Phase4模拟3-5件
```

### 专利家族管理

```
核心专利 A ──┬── 改进专利 A1, A2
             ├── 应用专利 A3(场景1), A4(场景2)
             └── PCT申请 A-PCT → US/EP/JP进入
```
