# 「饭搭子」小程序开发方案

> 从一顿饭到真朋友 · 线上走到线下，主打真诚

---

## 1. 项目概述

### 1.1 产品定位

微信小程序，面向企业内部员工的线下真诚社交平台。通过智能配对 + 七大社交场景，让同事从陌生人变成真正的朋友。

### 1.2 核心标签

- 线上到线下
- 主打真诚
- 跨部门连接
- 兴趣搭子
- 微信小程序

### 1.3 Slogan

**盲盒饭局 · 温暖夸夸会 · 兴趣搭子 · 拆书圈 · 剧本杀 · 逛市集**

---

## 2. 项目背景

### 2.1 部门墙壁垒

同部门观点和思维方式随时间趋同。跨部门交流能产生更大碰撞，但正式场合会被"部门墙""本位主义"阻碍。通过建立私下连接，交流质量更高。

### 2.2 线上社交浅表化

线上只能传递视觉和听觉，过滤了太多微妙情感信号。年轻人渴望的"真实感"在屏幕后无法安放。

### 2.3 线下社交新浪潮

"盲盒饭局""夸夸会""无料交换"——Z世代正在用创新方式重建线下连接。

### 2.4 连接—对话—实践

知识管理不是文档和搜索引擎。人与人之间的私下连接，质量更高、更有价值。随机午餐是第一个尝试，拆书帮是第二个。

```
Level 1 · 饭搭子 → Level 2 · 温暖搭子 → Level 3 · 兴趣搭子 → Level 4 · 真朋友
(Lunch Buddy)      (Warm Connection)      (Interest Buddy)      (Real Friend)
```

| 阶段 | 名称 | 说明 | 承载场景 |
|------|------|------|---------|
| L1 | 饭搭子 | 从一顿随机午餐/盲盒饭局打破0→1的陌生壁垒 | 盲盒饭局、1v1午餐 |
| L2 | 温暖搭子 | 通过夸夸会建立信任与温暖，从表面交流走向情感共鸣 | 夸夸会、感谢勋章 |
| L3 | 兴趣搭子 | 发现共同兴趣后组成圈子，从"聊得来"到"玩在一起" | 兴趣圈、拆书帮、协会 |
| L4 | 真朋友 | 相约剧本杀、逛市集、Hiking，从同事变成生活中的真朋友 | 剧本杀、逛市集 |

## 5. 七大社交场景

### 5.1 盲盒饭局

- **形式**：6人一桌，未知惊喜，主题午餐
- **流程**：填写匹配问卷 → 系统分桌 → 饭前不知同桌是谁
- **标签**：随机分桌、主题饭局

### 5.2 温暖夸夸会

- **形式**：1分钟找出对方3个优点，治愈社恐
- **特点**：没有竞争与比较，只有真诚赞美
- **标签**：真诚赞美、重新认识自我

### 5.3 兴趣搭子配对

- **形式**：选择兴趣标签（摄影/跑步/游戏/旅行/读书…），系统匹配同兴趣的跨部门伙伴
- **标签**：标签匹配、同频社交

### 5.4 拆书帮

- **形式**：3-8人共读圈，每周讨论，笔记沉淀为知识卡片
- **标签**：共读圈、知识卡片

### 5.5 兴趣协会

- **形式**：兴趣圈达到规模后升级为协会，有主页、活动日历和成员体系
- **标签**：协会主页、自治管理

### 5.6 相约剧本杀

- **形式**：饭后一键发起"续局"——剧本杀、密室逃脱、桌游夜
- **标签**：饭后续局、信任建立

### 5.7 逛市集 & 线下活动

- **形式**：周末逛创意市集、看展、Hiking、Citywalk
- **标签**：走出公司、真朋友

## 6. 核心功能模块

### 6.1 智能配对引擎（核心模块）

> 支持多种社交模式的智能匹配核心

- 1v1随机午餐：经典模式，跨部门优先
- 6人盲盒饭局：问卷匹配分桌，主题午餐
- 兴趣搭子匹配：基于标签精准配对
- 新员工Onboarding：新老员工分组配对，加速融入
- 地理位置感知 + 防重复 + 关系记忆 + 投诉拉黑

#### 6.1.1 配对算法要点

```
输入：用户标签、部门、地理位置、兴趣、历史配对记录、黑名单
规则：
  - 跨部门优先（不同部门权重更高）
  - 防重复（近N次不重复配对）
  - 黑名单过滤（投诉拉黑名单）
  - 地理位置就近（同园区/同城）
  - 兴趣匹配（兴趣标签交集权重）
输出：配对结果列表（1v1 或 6人桌分组）
```

### 6.2 温暖夸夸会系统

- 线下夸夸会报名与组织
- "1分钟3优点"计时互动
- 匿名夸夸卡：写下TA的闪光点
- 夸夸墙沉淀温暖赞美

### 6.3 破冰话题 & AI对话

- AI根据标签生成个性化话题
- 主题话题库：人生转折 / 超能力 / 最骄傲的事
- 饭局主题设定，优质话题社区沉淀

### 6.4 故事墙 & 饭后日记

- 饭后故事发布（图文/短视频）
- "最有料/最合拍/最巧合/最温暖"标签
- 互赠感谢勋章，月度精选最佳故事
- 活动结束自动推送问卷，闭环回访感受
- 双向评价 + 匿名安全反馈通道

### 6.5 兴趣圈 & 协会平台

- 一键创建/加入兴趣圈
- 圈子主页、成员管理、活动日历
- 达到阈值自动升级为协会

### 6.6 饭后续局 & 活动组队

- 饭后一键发起续局（剧本杀/桌游/密室）
- 周末活动推荐（市集/展览/Hiking）
- 一键组队，活动回忆录沉淀共同记忆

### 6.7 拆书帮 · 知识圈

- 3-8人共读圈，每周讨论
- 笔记沉淀为知识卡片
- 优秀圈子获"知识先锋"勋章

### 6.8 关系成就 & 连接图谱

- 个人连接图谱——跨越了多少部门墙？
- 关系进阶成就：饭搭子→真朋友
- 组织热力图 + 知识地图 + 活动大盘

## 7. 用户旅程

```
报名 → 配对 → 线下相遇 → 沉淀回馈 → 兴趣延伸 → 真朋友
```

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1. 报名 | 扫码进入、完善标签、选择场景 | 小程序扫码入口，填写基本信息和兴趣标签 |
| 2. 配对 | 智能匹配、推送通知、破冰话题 | 系统根据算法配对，发送订阅消息通知 |
| 3. 线下相遇 | 盲盒饭局、夸夸会、真诚交流 | 线下面对面社交 |
| 4. 沉淀回馈 | 故事墙、感谢勋章、夸夸卡 | 饭后反馈、故事发布、互赠勋章 |
| 5. 兴趣延伸 | 加入圈子、拆书帮、兴趣协会 | 基于兴趣深度连接 |
| 6. 真朋友 | 剧本杀、逛市集、生活好友 | 关系从同事延伸到生活 |

## 12. 风险与治理

### 12.1 已知风险

盲盒社交的不确定性既是魅力也是短板：可能遇到主导全场的饭友、点菜不顾分量与价格、饭后"私信轰炸"等。

### 12.2 治理机制

- 短期内被3人投诉 → 启动"拉黑"程序
- 持续收集反馈，动态更新匹配策略
- 饭后双向评价 + 匿名安全反馈通道
- 活动结束发送调查问卷，回访感受
- 设置"不再匹配"名单，尊重个人边界

### 12.3 设计原则

- **民间的、自愿的**：非公司官方行为，社区自愿发起
- **安全底线**：6人桌比1v1更安全，群体自我调节
- **渐进信任**：先饭局→再兴趣→最后私人活动
- **数据最小化**：仅采集匹配必需信息
