# ⚠️ 常见陷阱与避坑指南

> 标签：#lessons #pitfalls #debugging #experience
> 蒸馏来源：多个项目的实际踩坑经验
> 创建：2026-03-21 | 验证次数：多次

---

## 1. 文件操作陷阱

### 🔴 write_to_file 覆盖风险
- **场景**：对已有大文件使用 `write_to_file` → 可能截断或丢失内容
- **解决**：已有文件**必须先 read_file**，再用 `replace_in_file` 做精准修改

### 🔴 超长文件一次性生成
- **场景**：内容超过 token 限制 → 文件被截断
- **解决**：超过 500 行的内容，分成多个文件或分批追加写入

### 🟡 路径编码问题
- **场景**：中文路径/特殊字符 → 工具调用失败
- **解决**：优先使用英文命名，必要时用 URL 编码

## 2. 数据迁移陷阱

### 🔴 重复标题覆盖
- **场景**：批量转换时，同目录同标题的文件后者覆盖前者
- **解决**：转换前检测重复，加序号（如 `(2)`, `(3)`）区分
- **来源**：为知笔记迁移项目，328组重复标题导致364篇被覆盖

### 🟡 云端数据不可达
- **场景**：迁移工具假设本地有完整数据，但部分仅在云端
- **解决**：先统计本地缺失数量，提供云端补充下载方案

---

## 3. 多 Agent 设计陷阱

### 🔴 Agent 角色重叠
- **场景**：职责描述模糊 → 多个 Agent 做相同事情
- **解决**：用 RACI 矩阵检验，确保每个任务只有一个 Responsible

### 🟡 贯穿型 Agent 过多
- **场景**：每个 Phase 都有 5+ 个贯穿 Agent 参与 → 通信成本 O(n×m)
- **解决**：贯穿型 Agent ≤ 5 个，按「必须全程参与」严格筛选

### 🟡 缺少质量门控
- **场景**：Phase 流水线一路到底，没有中间检查点 → 错误累积放大
- **解决**：每个 Phase 结束设置 Gate，不合格则回退

---

## 4. 交互设计陷阱

### 🟡 一次性输出太多信息
- **场景**：用户提问简单，但回答列出 20+ 条详细信息 → 信息过载
- **解决**：先给摘要/概览，让用户选择深入哪个方向

### 🟡 忘记确认关键决策
- **场景**：自行决定了架构/方向/范围，未与用户确认
- **解决**：在方向性决策点使用 `ask_followup_question` 确认
